# LinkedIn Visual Content — Image & Carousel Text Skill

This skill generates the **exact text that appears on LinkedIn image and carousel slides** for Aiden AI. It is not about post captions — it is about what the viewer reads on the visual itself.

> Always load `aiden-ai-marketing-SKILL` alongside this skill for brand voice, value propositions, buyer personas, and Aiden AI product context.

---

## Part 1: The Golden Rules of LinkedIn Visual Text

These rules apply to EVERY piece of content produced by this skill. No exceptions.

### 1.1 The 5 Laws

**Law 1 — One idea per slide.**
If a slide is trying to say two things, it is saying nothing. Cut it to one.

**Law 2 — If it needs a full sentence, it's probably too long.**
Headlines. Fragments. Punchy statements. Ruthlessly cut connective tissue.

**Law 3 — The viewer spends 1.5 seconds per slide.**
Everything on that slide must land in 1.5 seconds. If it can't be scanned that fast, rewrite it.

**Law 4 — Contrast creates interest.**
"Most AI demos fail." > "AI can sometimes struggle in production environments."
Use contrast, tension, and surprise — not safe, descriptive language.

**Law 5 — The first slide wins the swipe or loses it.**
The cover slide is everything. If slide 1 doesn't create a pull to see slide 2, the carousel is dead.

---

### 1.2 Character & Word Limits Per Slide

| Element | Limit | Notes |
|---------|-------|-------|
| **Headline (main text)** | 4–8 words | Absolute max 10. Fewer is almost always better. |
| **Supporting line** | 8–12 words | Optional. One line only. Never two supporting lines. |
| **Label / eyebrow text** | 2–4 words | e.g., "Slide 3 of 6" or a category tag like "THE PROBLEM" |
| **CTA text (last slide)** | 5–8 words | One clear action. |
| **Stat / number callout** | The number + 3–5 word context | e.g., "10x faster to production" |

## Part 2: Single Image Posts

### 2.1 When to Use a Single Image

Use a single image when the message is **complete in one thought** — a strong stat, a bold claim, a short contrast, or a quote.

### 2.2 Single Image Structure

```
┌──────────────────────────────────────┐
│  [EYEBROW — optional, 2–4 words]     │
│                                      │
│  [HEADLINE — 4–8 words, max 10]      │
│                                      │
│  [SUPPORTING LINE — optional,        │
│   8–12 words max]                    │
│                                      │
│  [LOGO / BRAND MARK]                 │
└──────────────────────────────────────┘
```

### 2.3 Single Image Formulas

**Formula A — The Bold Claim**
```
EYEBROW:   [Category or context, 2–4 words]
HEADLINE:  [Provocative or contrarian statement]
SUPPORT:   [One line that adds credibility or specificity]
```
Example:
```
EYEBROW:   Enterprise AI
HEADLINE:  Most AI pilots never ship.
SUPPORT:   Aiden is built for production, not demos.
```

---

**Formula B — The Stat Callout**
```
HEADLINE:  [The number, styled large]
SUPPORT:   [What the number means, ≤ 10 words]
```
Example:
```
HEADLINE:  10x
SUPPORT:   Faster to production than building from scratch.
```

---

**Formula C — The Contrast**
```
LINE 1:    [The old/bad reality — 4–6 words]
DIVIDER:   vs.  /  →  /  not
LINE 2:    [The new/better reality — 4–6 words]
```
Example:
```
LINE 1:    6 months of AI infrastructure.
vs.
LINE 2:    6 weeks to production with Aiden.
```

---

**Formula D — The Quote / POV**
```
QUOTE:     "[First-person statement, 8–14 words]"
ATTR:      — Role or Persona (e.g., "— Every CTO, eventually")
```
Example:
```
QUOTE:     "We built the demo. We never built the product."
ATTR:      — Every AI team that skipped the runtime
```

## Part 3: Carousel Posts

### 3.1 Carousel Structure — The Arc

Every carousel must follow a **narrative arc**. The slides build on each other. It is not a random list of facts.

```
SLIDE 1 — HOOK        Create the pull. Tension, surprise, or bold claim.
SLIDE 2 — PROBLEM     Establish the pain. Make the reader nod.
SLIDE 3–N — CONTENT   Deliver the value. One insight per slide.
LAST SLIDE — CTA      Tell them exactly what to do next.
```

For **short carousels (3–5 slides):** Hook → 1–2 insight slides → CTA
For **medium carousels (6–8 slides):** Hook → Problem → 3–4 insight slides → CTA
For **long carousels (9–12 slides):** Hook → Problem → 5–8 insight slides → Summary → CTA

---

### 3.2 Slide-by-Slide Templates

---

#### SLIDE 1 — HOOK (Cover)

The most important slide. Must create **forward momentum** — a question, a tension, a bold claim that cannot be resolved without swiping.

**Hook formulas:**

```
TYPE A — The Provocative Claim
HEADLINE:  [Something surprising or contrarian, 4–8 words]
SUPPORT:   [One line that amplifies or contextualizes it]

Example:
"Your AI isn't getting smarter."
Here's why — and how to fix it.
```

```
TYPE B — The Number Hook
EYEBROW:   [Topic label]
HEADLINE:  [Surprising stat or number]
SUPPORT:   [What it means]

Example:
ENTERPRISE AI
70% of AI projects
never reach production.
```

```
TYPE C — The List Promise
HEADLINE:  [X things / reasons / signs / mistakes]
SUPPORT:   [What they'll walk away with]

Example:
5 signs your AI platform
isn't production-ready.
(and how to spot them fast)
```

```
TYPE D — The Direct Question
HEADLINE:  [Question that the reader has asked themselves]
SUPPORT:   [Promise to answer it in this carousel]

Example:
Why does your AI work in demos
but fail in production?
```

---

#### SLIDE 2 — PROBLEM (optional but powerful)

Name the pain clearly. Do not over-explain. One punchy statement.

```
LABEL:     THE PROBLEM  (or "SOUND FAMILIAR?" / "THE REALITY")
HEADLINE:  [The pain in 4–8 words]
SUPPORT:   [One line that makes it concrete or urgent]

Example:
SOUND FAMILIAR?
Your team spent 6 months
building infrastructure, not solutions.
```

---

#### SLIDES 3–N — CONTENT / INSIGHT SLIDES

Each slide = one idea. Structure:

```
LABEL:     [Slide number or category — e.g., "01" / "THE FIX" / "WHY IT MATTERS"]
HEADLINE:  [The insight or point, 4–8 words]
SUPPORT:   [One line expanding it, 8–12 words max]

Example:
01
Multi-model routing
means you're never locked in.
Route across Claude, GPT-4o, Gemini — automatically.
```

**Content slide patterns:**

| Pattern | Use When |
|---------|---------|
| **Insight + proof** | Sharing a platform capability with a specific metric |
| **Before / After** | Contrasting old way vs. Aiden way on one slide |
| **Short list (2–3 items)** | When the point has 2–3 sub-parts, keep each to 3–5 words |
| **Bold statement** | When the point stands alone — no support line needed |
| **Question + answer** | The headline is the question; the support line is the answer |

---

#### LAST SLIDE — CTA

One action. Clear and specific. No multiple CTAs.

```
HEADLINE:  [Action-oriented, 4–7 words]
SUPPORT:   [Where to go or what to do, 5–8 words]
CTA TEXT:  [Button-style text, 3–5 words]

Example:
Ready to ship production AI?
See how Aiden gets you there.
→ Book a demo
```

## Part 5: Voice Rules for Visual Text

These rules come directly from the Aiden AI brand voice (see `aiden-ai-marketing-SKILL`), adapted for the constraints of visual text.

### Do This

| Rule | Example |
|------|---------|
| Use sentence fragments | "Not a demo. A product." |
| Start with the tension | "Your AI isn't learning." not "Here at Aiden we believe AI should improve..." |
| Use numbers and specifics | "10x faster" not "significantly faster" |
| Use contrast and opposition | "POC ≠ Production" / "Demo vs. Reality" |
| Short words over long | "ship" not "deploy and release to production" |
| Second person | "Your AI" not "Enterprise AI systems" |
| Periods for punch | "It ships." / "It learns." / "It works." |

### Never Do This

| Avoid | Because |
|-------|---------|
| "Revolutionary" / "Game-changing" | Empty. Every competitor says it. |
| Full paragraphs on a slide | Nobody reads them. |
| More than one supporting line | Visual clutter. One line maximum. |
| Weak openers ("In today's world...") | Loses the viewer instantly. |
| Feature names without payoff | "MCP 2.0 native" alone means nothing. Add: "Every tool, every agent, instantly." |
| Passive voice | "AI is built by Aiden" → "Aiden builds AI that ships." |
| Ending on a vague CTA | "Learn more" → "Book a 30-min demo" |

## Part 7: Quick-Start Prompt Patterns

When a user gives a topic + 1–2 context lines, follow this process:

1. **Identify the format** — single image or carousel? If not specified, default to carousel for anything with more than one idea.
2. **Identify the persona** — who is this for? Use `aiden-ai-marketing-SKILL` personas to calibrate language.
3. **Choose the arc** — what's the tension on slide 1? What's the payoff on the last slide?
4. **Apply word limits ruthlessly** — write it long, then cut it by 50%.
5. **Read each slide in 1.5 seconds** — if you can't, it's too long.
6. **Output in the structured format** from Part 6.

---

> **Core principle**: LinkedIn visual text is not marketing copy compressed onto a slide. It is a completely different discipline — closer to billboard design than copywriting. Every word on a slide must earn its place. If it doesn't change the meaning, it doesn't belong. Write it tight, make it hit, and trust the visual designer to do the rest.
